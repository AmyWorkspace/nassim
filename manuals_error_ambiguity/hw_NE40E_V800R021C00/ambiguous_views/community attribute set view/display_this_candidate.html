display this candidate

Function

The display this candidate command displays the configurations that have not been committed in the current view.

Format

display this candidate

Parameters

None

Views

AS_Path set view, IPv4 address prefix set view, IPv6 address prefix set view, QPPB configuration group view, RD set view, Global variable set view, Community attribute set view, Route target set view, SoO set view, Route-filter policy view

Default Level

1: Monitoring level

Task Name and Operations

Task Name Operations
route-base read

Usage Guidelines

Usage Scenario

To check the configurations that have not been committed in the current view when configuring a complicated set, run the display this candidate command.

Example

The actual command output varies according to the device. The command output here is only an example.

# Display configurations that have not been committed in the IPv4 prefix set view.
<HUAWEI> system
[~HUAWEI] xpl ip-prefix-list ipv4-list1
[~HUAWEI-xpl-pfx] 1.1.1.0 24,
[~HUAWEI-xpl-pfx] 2.2.2.0 24,
[~HUAWEI-xpl-pfx] display this candidate
 xpl ip-prefix-list ipv4-list1
 1.1.1.0 24,
 2.2.2.0 24,
