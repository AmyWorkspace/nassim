abort

Function

The abort command cancels the configurations that have not been committed in the current view and returns to the system view.

Format

abort

Parameters

None

Views

AS_Path set view, IPv4 address prefix set view, IPv6 address prefix set view, QPPB configuration group view, RD set view, Global variable set view, Community attribute set view, Route target set view, SoO set view, Route-filter policy view

Default Level

2: Configuration level

Task Name and Operations

Task Name Operations
route-base write

Usage Guidelines

Usage Scenario

In case of incorrect configurations, you can run the abort command to cancel the configurations that have not been committed in the current view and return to the system view.

Example

# Configure an AS_Path set named as_path1 and cancel the configuration halfway.
<HUAWEI> system
[~HUAWEI] xpl as-path-list as_path1
[~HUAWEI-xpl-as] regular 1_2_3_4,
[~HUAWEI-xpl-as] abort
