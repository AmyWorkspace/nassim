l2-multicast pim router-port

Function

The l2-multicast pim router-port command configures a device to forward the PIM messages of a specific VLAN/VSI only to router interfaces.

The undo l2-multicast pim router-port command restores the default configuration.

By default, a device broadcasts received PIM messages, so that both router interfaces and non-router interfaces can receive PIM messages.

Format

l2-multicast pim router-port

undo l2-multicast pim router-port

Parameters

None

Views

VSI-AUTO view, VSI-BD view, VSI-DEFAULT view, VSI-STATIC view

Default Level

2: Configuration level

Task Name and Operations

Task Name Operations
l2mc write

Usage Guidelines

Usage Scenario

If a Layer 2 multicast device does need to forward PIM messages to its connected non-multicast devices, run the l2-multicast pim router-port command, so that the Layer 2 multicast device forwards PIM messages only to router interfaces. This configuration frees the connected non-multicast devices from processing PIM messages, reducing message forwarding and processing workloads and bandwidth consumption.

Prerequisites

IGMP snooping has been enabled using the igmp-snooping enable command in the system view and then in the VSI view.

Example

# Configure the device to forward the PIM messages of VSI vsi1 only to router interfaces.
<HUAWEI> system-view
[~HUAWEI] igmp-snooping enable
[*HUAWEI] mpls l2vpn
[*HUAWEI-l2vpn] quit
[*HUAWEI] vsi vsi1
[*HUAWEI-vsi1] pwsignal ldp
[*HUAWEI-vsi1-ldp] vsi-id 1
[*HUAWEI-vsi1-ldp] quit
[*HUAWEI-vsi1] igmp-snooping enable
[*HUAWEI-vsi1] l2-multicast pim router-port
