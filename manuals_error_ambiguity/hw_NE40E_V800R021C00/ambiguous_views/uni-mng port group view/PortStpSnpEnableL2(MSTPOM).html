stp tc-snooping enable

Function

The stp tc-snooping enable command enables an interface to snoop STP TC BPDUs.

The undo stp tc-snooping enable command disables an interface from snooping STP TC BPDUs.

By default, an interface is not enabled to snoop STP TC BPDUs.

Format

stp tc-snooping enable

undo stp tc-snooping enable

Parameters

None

Views

Layer 2 100GE interface view, Layer 2 10GE interface view, 25GE-L2 view, 400GE-L2 view, Layer 2 40GE interface view, Layer 2 50GE interface view, Layer 2 50|100GE interface view, Layer 2 Eth-Trunk interface view, Layer 2 GE interface view, UNI-MNG port group view, Interface group view

Default Level

2: Configuration level

Task Name and Operations

Task Name Operations
mstp write

Usage Guidelines

Usage Scenario

When the topology of an Ethernet ring connected to an EVPN changes, devices on the EVPN need to be informed of the topology change in time. After this command is run on an interface of a device that accesses the EVPN, the interface can snoop STP TC BPDUs in time to detect the topology change of the Ethernet ring, so as to instruct BD-bound sub-interfaces to update ARP and MAC entries.

Prerequisites

STP and other loop prevention protocols have been disabled on the interface where STP TC BPDU snooping is to be enabled.

The process configurations and instance-VLAN mapping configurations on the device where STP TC BPDU snooping is to be enabled must be the same as those on other devices on the ring.

Example

# Enable GE 1/0/1 to snoop STP TC BPDUs.
<HUAWEI> system-view
[~HUAWEI] interface GigabitEthernet 1/0/1
[~HUAWEI-GigabitEthernet1/0/1] portswitch
[~HUAWEI-GigabitEthernet1/0/1] stp disable
[~HUAWEI-GigabitEthernet1/0/1] stp tc-snooping enable
