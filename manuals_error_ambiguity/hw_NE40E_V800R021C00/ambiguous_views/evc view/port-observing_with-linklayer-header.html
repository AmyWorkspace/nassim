port-observing with-linklayer-header

Function

The port-observing with-linklayer-header command configures an observing port to mirror packets including the link layer header.

The undo port-observing with-linklayer-header command disables the function of mirroring the packet from the link layer header and the mirrored packet is directly sent by the observing port.

By default, an observing port mirrors packets excluding the link layer header.

Format

port-observing with-linklayer-header

undo port-observing with-linklayer-header

Parameters

None

Views

EVC view, Eth-Trunk sub-interface view, GE sub-interface view, EVC view of an Eth-Trunk interface, EVC view of an Ethernet interface, Port extension sub-interface view

Default Level

3: Management level

Task Name and Operations

Task Name Operations
mirror write

Usage Guidelines

Usage Scenario

To configure an observing port to mirror packets including the link layer header, run the port-observing with-linklayer-header command on the observing port.

Prerequisites

An observing port has been configured.

Precautions

Only sub-interfaces support this command.

When a main interface is used as the observing port for local mirroring, you do not need to run this command. The observing port mirrors packets including their link layer headers.

Example

# Configure the observing port GE 1/0/0.1 to mirror packets including their link layer headers.
<HUAWEI> system-view
[~HUAWEI] interface GigabitEthernet1/0/0.1
[~HUAWEI-GigabitEthernet1/0/0.1] port-observing observe-index 1
[*HUAWEI-GigabitEthernet1/0/0.1] port-observing with-linklayer-header
