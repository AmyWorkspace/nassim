end-list

Function

The end-list command concludes the configuration of a set, exits the set view, and returns to the system view.

By default, the end-list command is not run.

Format

end-list

Parameters

None

Views

AS_Path set view, IPv4 address prefix set view, IPv6 address prefix set view, RD set view, Community attribute set view, Route target set view, SoO set view

Default Level

2: Configuration level

Task Name and Operations

Task Name Operations
route-base write

Usage Guidelines

Usage Scenario

Route-filters are used to filter routes based on sets or a single element. To conclude the configuration of a set, exit the set view, and return to the system view, run the end-list command.

Prerequisites

A name and elements have been configured for a set.

Precautions

A set is only a group of data used as matching rules and does not have the permit or deny function. Therefore, a set can be used to filter routes only after it is referenced by a route-filter.

Example

# Configure an AS_Path set named as_path1.
<HUAWEI> system
[~HUAWEI] xpl as-path-list as_path1
[~HUAWEI-xpl-as] regular 1_2_3_4,
[~HUAWEI-xpl-as] regular ^200$,
[~HUAWEI-xpl-as] regular 100
[~HUAWEI-xpl-as] end-list
