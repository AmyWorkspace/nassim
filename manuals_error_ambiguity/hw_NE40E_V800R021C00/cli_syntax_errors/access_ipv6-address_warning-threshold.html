access ipv6-address warning-threshold

Function

The access ipv6-address warning-threshold command configures upper and lower IPv6 address alarm thresholds for a device.

The undo access ipv6-address warning-threshold command restores the default configuration.

By default, the IPv6 address usage threshold-crossing alarm is not enabled on a device.

Format

access ipv6-address warning-threshold upper-limit upper-limit-val lower-limit lower-limit-val

undo access ipv6-address warning-threshold upper-limit | lower-limit

Parameters

Parameter Description Value
lower-limit lower-limit-val

Specifies a lower alarm threshold.

The value is an integer ranging from 0 to 99.
upper-limit upper-limit-val

Specifies an upper alarm threshold.

The value is an integer ranging from 1 to 100.

Views

System view

Default Level

2: Configuration level

Task Name and Operations

Task Name Operations
aaa-access write

Usage Guidelines

Usage Scenario

To configure upper and lower IPv6 address alarm thresholds, run the access ipv6-address warning-threshold command. When the percentage of the number of IPv6 addresses of access users on a device in the device's IPv6 specification (1M) exceeds the upper alarm threshold or goes below the lower alarm threshold, the device generates an alarm.

Precautions

In VS mode, this command is supported only by the admin VS.

Example

# Set the upper and lower IPv6 address alarm thresholds of a device to 80% and 20%, respectively.
<HUAWEI> system-view
[~HUAWEI] access ipv6-address warning-threshold upper-limit 80 lower-limit 20
