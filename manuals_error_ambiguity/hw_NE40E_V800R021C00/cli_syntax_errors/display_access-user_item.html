display access-user item

Function

The display access-user item command displays information about all access users.

Format

Parameters

Parameter Description Value
include

Match the users with the specified string.

-
include-text

Include information.

The value is in text format. The value ranges from 1 to 200.
circuit-id

Circuit ID.

-
username username

Displays information about the access users with specified user name.

The value is a string of 1 to 253 characters.
match-quote-context

Displays information about access users whose user names contain spaces.

If user names contain spaces, enclose the user names with double quotation marks ("") to carry the.

match-quote-context keyword and check user information based on the user names.

The.

match-quote-context keyword can be associated only when user names are enclosed using double quotation marks ("").

-
accounting-state

Displays information about online users based on a specified accounting state.

  • idle: Displays information about online users in the idle accounting state.
  • ready: Displays information about online users in the ready accounting state.
  • accounting: Displays information about online users for whom accounting is being performed.
  • leaving-flow-query: Displays information about online users whose offline traffic is being queried during the logout process.
  • wait-acct-start: Displays information about online users whose offline traffic is being queried during the logout process.
  • wait-acct-stop: Displays information about online users waiting for an Accounting Stop response from the RADIUS server.

Leaving-flow-query.

-

Wait accounting stop.

-

Displays MAP-T user information.

-

Displays MAP-E user information.

-

-

-
cgn-centralized-user

Displays CGN centralized user information.

-

Displays information about the access users who use the specified Radius server group.

The value is a string of 1 to 32 characters, including characters A to Z and a to z (case-sensitive), numerals (0 to 9), punctuation mark (.), dash (-), and underline (_).
web-auth-server web-auth-server

Displays information about the access users who are authenticated by the web authentication server with a specified IPv4 address.

The value is in dotted decimal notation.
web-auth-server6

Web-auth-server ipv6 address.

The value is a 32-digit hexadecimal number, in the format of X:X:X:X:X:X:X:X.

User traffic.

overflow

-
ipv4

-

-
inbound

-
above

Above.

-
above rate-inbound-above-value

Displays the users whose real-time upstream traffic rate is greater than or equal to the specified rate.

The value is an integer ranging from 0 to 4294967295, in kbyte/min. When ui-mode type2 is configured, the unit is kbit/s.
above rate-outbound-above-value

Displays the users whose real-time downstream traffic rate is greater than or equal to the specified rate.

The value is an integer ranging from 0 to 4294967295, in kbyte/min. When ui-mode type2 is configured, the unit is kbit/s.
value-high

Value of high.

The value is an integer ranging from 0 to 4294967295.
value-low

Value of low.

The value is an integer ranging from 0 to 4294967295.
below

Below.

-
below rate-inbound-below-value

Displays the users whose real-time upstream traffic rate is less than the specified rate.

The value is an integer ranging from 0 to 4294967295, in kbyte/min. When ui-mode type2 is configured, the unit is kbit/s.
below rate-outbound-below-value

Displays the users whose real-time downstream traffic rate is less than the specified rate.

The value is an integer ranging from 0 to 4294967295, in kbyte/min. When ui-mode type2 is configured, the unit is kbit/s.
outbound

Specifies outbound direction.

-
service-traffic

Service traffic.

-
edsg

EDSG.

light-traffic-user

Displays information about users whose traffic is light in many statistics collection periods.

both

-

Interval.

-
bod

BOD.

-
daa

Displays information about users with a destination address accounting service type.

-
service-type

Service type.

-
service-policy service-policy-name

Displays information about users with a specified service policy.

The value is a string of 1 to 200 case-sensitive characters, spaces not supported.
slot slot-id

Specifies the ID of the slot where a board resides.

The value is a string of 1 to 15 case-sensitive characters, spaces not supported.
card card-number

Specifies a subcard ID. The subcard ID range is determined by the device model.

The value is a string of 0 to 65535 case-sensitive characters, spaces not supported.
domain domain-name

Displays information about online users in the specified domain.

The value is a string of 1 to 64 case-insensitive characters.

Displays information about online users in a specified mapping domain.

The value is a string of 1 to 64 case-insensitive characters.
vpn-instance vpn-instance-name

Displays information about online user bound to the specified VPN instance.

The value is a string of 1 to 31 case-sensitive characters. If spaces are used, the string must start and end with double quotation marks (").

Displays information about online users who use the specified LAC tunnel to access the LNS.

The value is a string of 1 to 253 characters.
lac-ip-address lac-ip-address

Displays information about online users on the tunnel with a specified source IP address on the LNS.

The value is in dotted decimal notation.
time time-number

Displays information about the online users in a specified time period.

The value ranges from 1 to 7200, in seconds.
option37

Qos profile.

Usage Guidelines

Usage Scenario

To view the information about the access users or locate the user access fault, you can run the display access-user command to view the information about all access users. You can also view information about access users on the device by specifying the domain name, interface name, user name, and so on.

In detailed information about access users, (Radius) is displayed for each attribute delivered by the RADIUS server.

Precautions

The display access-user light-traffic-user command displays information about users with abnormal traffic. In the physical RUI scenario, the traffic statistics reported by the backup device is always 0. Therefore, all users on the backup device are users with abnormal traffic. As such, you cannot run this command to query information about users based on actual user traffic.
